一、我的人生目标到底是什么？

  • 成为一名在领域内享有盛名的优秀技术人。在大公司内担任架构师，统筹整个项目，并追求更卓越的发展。
  • 成立自己的创业公司，招揽志同道合的小伙伴们，为用户提供优质的技术服务。
  • 要有一定的影响力，在输入的同时不断分享，持续输出。我要出版自己的技术书籍。
  • 除了自身，还要让家庭变得更好，无论是物质上还是精神上，都要富足，让家人过上更好的生活。
  • 游历更多的地方。世界那么大，我也想去走走。

二、如何度过今后三年？

  • 技能：还剩两年的研究生阶段，要在 2017 年内快速夯实巩固基础，努力提升各方面技能，为就业做准备。
  • 科研： 科研项目不能落下，横向项目和纵向项目要同步进行，要有适当充分的准备和进展突破，在科研汇报中游刃有余。
  • 毕业：优秀地完成硕士毕业论文。
  • 实战：科研学习之余，做更多的项目，丰富 Github 仓库，坚持博客写作，坚持分享。
  • 内功：2017 下半年参加 PAT 考试，在数据结构和算法上有更深入的理解，提升程序员的内功。
  • 实习：前端实习，时间精力充裕，还可以接点私活。
  • 工作：争取毕业之后进入阿里巴巴，从事前端开发，在深入与掌握前端各个方面的同时，在技术的深度和广度进行挖掘，提升技能，拓展技术栈，努力成为全栈工程师。
  • 健康：当然，努力学习工作的同时，还要有健康的身体，劳逸结合，追求的不是高效率，而是高效能。

三、如果现在人生只剩 6 个月？

乔布斯每天醒来经常问自己，如果这是我生命的最后一天，我会以怎样的方式度过。问自己这样的问题，可以找出那些对自己非常重要的但是经常觉得来不及做的事。

  • 最先想到的是感恩。如果人生只剩 6 个月，我可能不会把提升自己放在首位，而是更加关注家人们，关注一路走来，那些给予我帮助的人，那些一直陪伴着我的人，那些一起玩耍奋斗的小伙伴。
  • 不忘初心，我还是会做点什么，而不是坐吃享乐，等待死神的到来。我会偶尔画些画，做些手工（这都是我小时候最爱的做的事情），当然，我也会结合自己小时候的兴趣和现在的技能，做一些有意思的网站或者 App，或者是纯属娱乐，或者是能帮助到大家的公益项目。
  • 我会写自传。我要趁着我记忆犹在，把我的人生记录下来，从童年，到中学大学，我怎样一路走来，我遇到哪些人，发生哪些事，我某个时刻在想些什么，有什么感悟，能回忆起来的我都会记下来。哪怕没人看，我觉得记录下来，人走的时候，会少些遗憾。
  • 技术人生，我还是会学习和分享技术，和志同道合的小伙伴交流，或许这没有什么用（你还想靠此进入大公司工作？就算进了，留给你的时间还多吗？），但是，人生总得有些不问目的，没有功利，单纯就是想去做的事情。

写下来发现：人到生命尽头，就不会那么得自私，而是想多地想到别人，想到过去美好的事情。

将上述所列的人生目标进行优先级排序，选出 3 个最重要的目标，分别标记为 A-1， A-2， A-3。

四、我的 3 个最重要的长期目标是：

  • A-1： 成为一名在领域内享有盛名的优秀技术人。在大公司内担任架构师，统筹整个项目，并追求更卓越的发展。
  • A-2： 要让家庭变得更好，无论是物质上还是精神上，都要富足，让家人过上更好的生活。
  • A-3： 成立自己的创业公司，招揽志同道合的小伙伴们，为用户提供优质的技术服务。

五、实现长期目标所要做的活动

（一）A-1 目标

A-1： 成为一名在领域内享有盛名的优秀技术人。在大公司内担任架构师，统筹整个项目，并追求更卓越的发展。

1. 前端工程师

  1. 快速掌握 HTML + CSS + JS，阅读相关技术书籍，并通过视频进行学习。
  2. 每天花一定时间学习 JS 的语法和 JS 的面向对象知识，用 JS 开发小网站，做各种特效。
  3. 用 JS 等前端技术，利用 HBuilder 学习 App 开发，每天要多精进一点，要将其在前端的应用与 App 开发融汇贯通。关于 HBuilder 的学习，要坚持，比如每天要掌握一些基础的 MUI 控件，逐渐学会 H5+ 的运行机理，并在实战中总结这些经验。
  4. JQuery 的学习，在对 JS 有一定的掌握之后，就要上手 JQuery 。
  5. 学习前端开发的牛人，要多看他们的经验分享和技术分享，不能埋头苦干，时刻知道自己当前所处的阶段。
  6. 钻研前人项目，前端也有很多开源的项目和一些比较美观的网站，尝试着去理清他们的实现机理。
  7. 个人博客目前采用的是 hexo + material，material 主题非常美观，但是作为一个前端工程师，怎么能停留在一直使用他人主题呢。所以，要开发自己的主题，从中学习视觉、实现上的技巧，并对自己的 github 主题项目写好中英文 readme 文档，并搭建官方网站对主题进行介绍。
  8. 多阅读他人的博客，哪怕你觉得他不是牛人，但是只要对你有帮助，那就应该虚心学习。

2. 全栈

成为一名合格的架构师，首先要是一名全栈工程师。全栈意味着你不能只是掌握前端开发技术。除此之外

  1. 深入的算法能力。2017 年暑假每天要能够钻研数据结构和算法，自己动手写实现，并去做历年的 PAT 甲级试题，不但培养自己的算法能力，还能提高自己的编程能力。通过下半年的 PAT 甲级考试来检验自己的学习水平。
  2. 软件开发能力。要有优秀的面向对象设计能力，这要求对于设计模式，面向对象设计原则要有充分的理解和掌握，要学会用 UML 图来表达和交流自己的想法。
  3. 后端能力。学会 java 和 node.js 开发后端，要掌握并发编程和网络编程；理解计算机网络（要买本网络方面的书来夯实基础），要能够独立为网站或 App 开发后端。
  4. 数据库和 Web 安全。这些方面有待进一步的学习。

3. 架构师

在达到优秀前端工程师之前，对架构师要有基本的了解，多读那些优秀架构师的故事，学习他们的经验，要知道架构师要掌握哪些能力等等。
